# Style Imitate — 风格模仿与AI痕迹清除

## 任务目标

阅读用户存放在 `~/works/阐述/` 下的写作样本（中英文均可），提取用户独特的写作风格特征，然后严格按照该风格完成写作任务。核心要求：**产出的文本必须让人看不出是AI写的**——没有ChatGPT/Claude特有的句式习惯。

## 执行流程

### Step 1：读取全部写作样本

```bash
ls ~/works/阐述/*.md
```

读取每个 `.md` 文件的开头部分，确保覆盖足够多的样本以获得稳定的风格画像。如果 `.docx` 文件存在且需要参考，尝试用 `pandoc` 或 `python-docx` 提取文本。

### Step 2：构建风格画像

从样本中提取以下风格维度：

#### 2.1 句式特征
- 句子长度分布（偏好长句还是短句？长短如何交替？）
- 段落长度（一段几句话？）
- 是否频繁使用破折号、冒号、分号？
- 是否有特定句式偏好（如排比、反问、设问）？

#### 2.2 结构特征
- 章节标题的格式（"一、二、三" vs "1. 2. 3." vs "###"）
- 是否使用表格作为论述工具？
- 段落之间如何过渡（完全不使用过渡词？用空行分隔？用"但是/不过"自然衔接？）
- 文章开头方式（直接切入主题 vs 设置背景 vs 从具体问题出发？）

#### 2.3 措辞特征
- 学术化程度（术语密度、英语术语的使用频率）
- 是否使用"笔者认为"/"本文认为"这类限定语？
- 是否使用比喻或意象？
- 第一人称的使用频率和方式

#### 2.4 语气特征
- 确定性程度（直接断言 vs 温和限定）
- 是否有幽默/讽刺/冷峻的口吻？
- 情感表达方式（克制还是外露？）

#### 2.5 中英混用模式
- 何时插入英文术语？（仅第一次出现时标注？持续使用英文原文？用括号括注？）
- 是否在中文行文中直接嵌入英文短语或句子？

### Step 3：对照 AI 痕迹清单，逐项清除

**以下句式必须在最终输出中完全消除：**

#### 句式层面的 AI 痕迹
1. ❌ "不是……而是……" → 直接说"是X"或分两句说
2. ❌ "不仅……而且……" → 删掉"不仅"，只说"而且"部分，或用"也"
3. ❌ "值得注意的是" / "值得强调的是" → 直接陈述内容，不加评注
4. ❌ "不可否认的是" / "毫无疑问" → 直接断言或删掉
5. ❌ "从某种意义上说" / "在某种程度上" → 删掉，或说"部分在于"
6. ❌ "需要指出的是" / "需要说明的是" → 删掉"需要"，直接说
7. ❌ "值得一提的是" → 删掉
8. ❌ "正如（前文/上文）所述" → 直接说内容，不要回指
9. ❌ "如下所示" / "如上所述" / "如前所述" → 删掉或改用"见下表"
10. ❌ "综上所述" / "总而言之" / "总的来说" → 直接归纳，不加帽子
11. ❌ "首先/其次/最后" 作为段落开头 → 用数字序号自然过渡，或直接分段
12. ❌ "这意味着" / "这也就意味着" / "换言之" → 删掉，直接说结论
13. ❌ "不仅如此" / "更为重要的是" → 删掉，直接说第二个点
14. ❌ "从这个角度来说" → 删掉
15. ❌ "这一现象/这一问题" 泛化指代 → 明确指代对象，或用"这"一笔带过
16. ❌ "引发了一个重要问题" / "引出了一个值得思考的问题" → 直接问问题，不加前戏

#### 破折号使用规范
17. ❌ 每100字使用超过1个破折号（除非原文样本中明确如此）
18. ❌ 用破折号替代逗号、冒号或括号 → 优先使用冒号或逗号
19. ❌ 句末用破折号制造"悬念"或"省略"效果

> **破折号的替代方案：**
> - 解释说明 → 冒号或逗号
> - 插入语 → 逗号或括号
> - 强调 → 直接调整语序
> - 转折 → "但/不过/然而"

#### 段落与衔接的 AI 痕迹
20. ❌ 每段末尾一句做总结/概括 → 让段落自然结束
21. ❌ 段落之间使用相同的过渡模式 → 避免"然而→然而→然而"式开篇
22. ❌ 每段都有"帽子句"（topic sentence + 展开的机械模式） → 允许段落自然起止
23. ❌ 过度使用"但是/然而/不过"作为段落开头 → 减少到每2-3段最多一个

#### 措辞层面的 AI 痕迹
24. ❌ 堆砌形容词（深刻的、重要的、关键的、显著的等） → 只用"必要"的词
25. ❌ 用大词替代小词（"实施"替代"做"、"探究"替代"研究"、"是否具备xx性"替代"能不能"）
26. ❌ 高频使用"性/化/度/力/论"字尾抽象名词（可能性、有效性、系统性、多元化……）
27. ❌ 过度使用"在……中/在……下/在……上"作为前置状语
28. ❌ "体现了"/"展现了"/"呈现出" → 能用"是"就用"是"
29. ❌ "这一xxx"（这一过程、这一转变、这一趋势） → 直接说"这"或具体主语

#### 段落长度规范
30. ❌ 连续三个段落长度大致相等 → 让段落长短自然参差
31. ❌ 段落全部超过5行 → 必要时使用一两句话的短段

### Step 4：写作执行

1. 根据 Step 2 的风格画像，撰写内容
2. 每写完一段，对照 Step 3 检查是否有 AI 痕迹
3. 如果发现 AI 句式，改写直到自然
4. 初稿完成后，全文扫描 AI 痕迹并清除

### Step 5：终检清单

- [ ] 全文没有"不是……而是……"
- [ ] 全文没有"值得注意的是"类评注语
- [ ] 全文没有"首先/其次/最后"机械框架
- [ ] 全文没有"综上所述"类总结帽
- [ ] 破折号数量控制在合理范围（对照样本）
- [ ] 没有堆砌形容词
- [ ] 没有"在……中/下/上"过度使用
- [ ] 段落长短不整齐
- [ ] 不需要的"性/化/度/力"字尾已清除
- [ ] 读起来不像AI写的

## 注意事项

1. **不要过度修正**：用户的风格本身可能包含破折号（如学术论文中引用文献时的——）。关键是"像用户写的"，不是"符合某个完美标准"。
2. **当语言=zh时**：中文写作避免西化句式。用户的中文风格是自然的中文，不翻译腔。检查是否有英文语法结构被直译到中文。
3. **当语言=en时**：英文写作时注意用户偏好的学术英语风格（直接、简洁、不啰嗦）。
4. **表格是用户常用的工具**：当需要对比时，优先考虑是否可以用表格呈现。表格应简洁，不要装饰过度。
5. **用具体替代抽象**：用户倾向于用具体例子支撑论点，而不是空泛讨论。保持这个习惯。
6. **原文样本中的特殊风格**：如果用户样本显示某种独特习惯（如编号系统、特殊标点用法），继承这些习惯。
7. **不要"过于完美"**：AI写作的一个标志是每句话都是"对"的、每一处都无懈可击。人类写作有自然的节奏变化、偶尔的碎片化表达。允许一定的不规整。
8. **中英混用**：如果用户样本中频繁中英混用，保持这一特征；如果样本中英文术语都加了中文翻译或注释，也保持。
